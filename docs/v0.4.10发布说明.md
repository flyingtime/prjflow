# v0.4.10 发布说明

**发布日期**: 2025年12月02日

## 版本概述

v0.4.10 版本是一个重要的用户体验优化版本，统一了所有管理模块的弹窗详情和非弹窗详情内容，确保用户在弹窗和详情页面中看到的信息完全一致。本次更新主要针对项目、任务、版本等模块的弹窗详情进行了完善，添加了统计信息、历史记录、关联数据等完整内容，并补充了项目弹窗详情顶部操作按钮，提升了用户体验的一致性和完整性。同时优化了前端构建配置，改进了代码分割策略，减少了构建警告。

### 7. 前端构建配置优化 🛠️

#### 功能描述
优化了前端构建配置，改进了代码分割策略，将大型库分离到独立的 chunk，减少主 chunk 大小，提升加载性能。

#### 优化内容
- **智能代码分割**：使用函数形式的 `manualChunks`，按库类型自动分块
- **库分离策略**：
  - `vue-core` - Vue 核心库（单独）
  - `vue-router` - Vue Router（单独）
  - `pinia` - Pinia 状态管理（单独）
  - `ant-design-vue` - Ant Design Vue UI 库（单独）
  - `ant-design-icons` - Ant Design 图标库（单独）
  - `echarts` - ECharts 图表库（单独）
  - `highlight` - 代码高亮库（单独）
  - `marked` - Markdown 解析库（单独）
  - `dayjs` - 日期处理库（单独）
  - `axios` - HTTP 客户端库（单独）
  - `qrcode` - 二维码生成库（单独）
  - `vendor` - 其他 node_modules 依赖

#### 功能特点
- **更细粒度分割**：每个大型库独立成 chunk，便于缓存和按需加载
- **更好的缓存策略**：单个库更新时只影响对应 chunk
- **更快的首屏加载**：主 chunk 更小，加载更快
- **合理的警告限制**：将 `chunkSizeWarningLimit` 设置为 2000KB，避免对大型库（如 Vue 3 运行时、highlight.js、echarts）产生不必要的警告

#### 技术实现
```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks(id) {
        if (id.includes('node_modules')) {
          // 按库类型分离
          if (id.includes('vue-router')) return 'vue-router'
          if (id.includes('pinia')) return 'pinia'
          if (id.includes('vue') && !id.includes('vue-echarts')) return 'vue-core'
          // ... 其他库分离逻辑
        }
      }
    }
  },
  chunkSizeWarningLimit: 2000
}
```

#### 优化效果
- **构建警告减少**：合理的警告限制，避免对大型库产生警告
- **加载性能提升**：更小的主 chunk，更快的首屏加载
- **缓存效率提升**：库更新时只更新对应 chunk
- **并行加载**：多个小 chunk 可以并行加载

#### 涉及的模块
- ✅ **构建配置** (`frontend/vite.config.ts`) - 优化代码分割策略和警告限制
